# Reminder Message Reference

This file documents the current built-in reminder text generated by the app.

Why the older version showed separated last and first names:
- The older reference used a separator between last name and first name from an earlier display preference.
- The reminder messages now use normal readable names like `Sadhu Aksharvandandas`.
- Passenger and driver phone numbers should appear anywhere those people are listed in the built-in reminder text.

Runtime sources:
- Built-in reminder builders: `packages/data/src/lib/repository.ts`
- Google flight-status link builder: `packages/core/src/lib/format.ts`
- Rule-based reminders still use saved templates from the Reminders UI/API

## 1. Built-In Flight Reminder Format

Used by:
- flight added / changed / cancelled notifications
- 48 hour flight reminders
- one-off `Send reminder now` fallback when no custom text is entered

Current generated structure:

```text
Jai Swaminarayan,

{changeLabel}

Flight details:

Route: {departure_airport} → {arrival_airport}
Flight: {flight_number}

Departure: {departure_date_time}
Arrival: {arrival_date_time}

Passengers:
{passenger_1_name} - {passenger_1_phone}
{passenger_2_name} - {passenger_2_phone}

Transport Sevak:
Pickup - {pickup_driver_name} ({pickup_driver_phone})
Dropoff - {dropoff_driver_name} ({dropoff_driver_phone})

Accommodation:
{accommodation}

If you have any questions, reach out to {coordinator_name} - {coordinator_phone}

Link to track the flight status:
{google_flight_status_url}
```

Notes:
- `Passengers:` shows all passengers attached to the itinerary.
- `Transport Sevak:` shows all assigned drivers across pickup/dropoff tasks.
- `Accommodation:` falls back to `Not set` when there is no accommodation text.
- The coordinator line falls back to a generic sentence if no coordinator contact is available.
- The Google link is generated from airline code + flight number + departure date + departure time.

Example:

```text
Jai Swaminarayan,

48 hour flight reminder

Flight details:

Route: SFO → PHX
Flight: UA1936

Departure: Apr 28, 2026 8:30 AM
Arrival: Apr 28, 2026 10:45 AM

Passengers:
Sadhu Aksharvandandas - 555-111-2222
Sadhu Neelkanthdas - 555-333-4444

Transport Sevak:
Pickup - Mahesh Patel (555-999-0000)
Dropoff - Rakesh Patel (555-888-7777)

Accommodation:
Phoenix Mandir guest room

If you have any questions, reach out to West Santo Travel Coordinator - 555-222-3333

Link to track the flight status:
https://www.google.com/search?q=UA1936%20flight%20status%202026-04-28%2008%3A30
```

## 2. Built-In Transport Reminder Format

Used by:
- transport assignment / change / cancellation notifications
- 6 hour pickup reminders
- 6 hour dropoff reminders

Current generated structure:

```text
Jai Swaminarayan,

{changeLabel}

Transport details:

Task: {PICKUP_or_DROPOFF}
Airport: {airport_code}
Scheduled: {scheduled_date_time}

Passengers:
{passenger_1_name} - {passenger_1_phone}
{passenger_2_name} - {passenger_2_phone}

Transport Sevak:
Pickup - {driver_name} ({driver_phone})

If you have any questions, reach out to {coordinator_name} - {coordinator_phone}
```

Notes:
- Passenger and driver numbers are included when available.
- If there are no assigned drivers yet, the section shows `None`.

## 3. One-Off "Send Reminder Now"

UI location:
- `/reminders` -> `Send flight reminder now`

Behavior:
- If a custom message is entered, that exact text is sent.
- If the custom message box is blank, the app falls back to the built-in flight reminder format from section 1 with `changeLabel = Flight reminder`.
- The one-off sender also sends the same generated text to assigned drivers when SMS is selected.

## 4. Google Flight-Status Link Builder

Utility:
- `buildGoogleFlightStatusUrl(...)`
- file: `packages/core/src/lib/format.ts`

Input:
- `airlineCode`, example: `AA`
- `flightNumber`, example: `2282`
- `departureDate`, example: `2026-04-28`
- optional `departureTime`, example: `08:30`

Behavior:
- Combines airline code + flight number into one value like `AA2282`
- Builds a Google Search query in this format:
  - `AA2282 flight status 2026-04-28 08:30`
- Omits departure time when it is missing
- Uses URL encoding and returns only the URL string

Example output:

```text
https://www.google.com/search?q=AA2282%20flight%20status%202026-04-28%2008%3A30
```

## 5. Rule-Based Reminder Templates

Rule-based reminders still use saved template text. They do not automatically switch to the built-in format above.

Placeholder syntax:

```text
{{ variable_name }}
```

Flight departure rule variables:
- `{{passenger_name}}`
- `{{flight_number}}`
- `{{departure_airport}}`
- `{{arrival_airport}}`

Pickup scheduled rule variables:
- `{{driver_name}}`
- `{{airport_code}}`
- `{{mandir_name}}`
- `{{task_type}}`

## 6. Where To Change The Real Copy

If you want to change the actual outgoing built-in reminder text later, update:
- `packages/data/src/lib/repository.ts`
  - `buildFlightSummaryText(...)`
  - `buildTransportSummaryText(...)`
  - `queueOneOffFlightReminder(...)`

If you want to change the Google flight-status link behavior, update:
- `packages/core/src/lib/format.ts`
  - `buildGoogleFlightStatusUrl(...)`

If you want to change rule-based reminder copy, update the saved reminder-rule templates from the Reminders UI or API.
